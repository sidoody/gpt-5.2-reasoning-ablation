# Security Policy

## Supported Versions

Security fixes are applied to the latest state of the default branch.

## Reporting a Vulnerability

Please do not open public issues for suspected vulnerabilities.

Instead, report privately with:

- a clear description of the issue
- reproduction steps or proof of concept
- potential impact
- any suggested mitigation

I will acknowledge receipt as quickly as possible and follow up with next steps.

## Scope Notes

This repository is a research benchmark harness and is not intended as a clinical decision system.
Even so, reports of security, data handling, and dependency risks are welcome.
